# WorkmanWise — Brand Master Specification

> This document is the single source of truth for brand execution.
> If you are an AI generating content, a designer building pages, or a human writing copy — this is how you think.

---

## 1. Brand Operating System

### Core Worldview
- You don't need more software. You need fewer things done right.
- Manufacturing isn't broken — it's fragmented. Most problems are handoff problems.
- Automation is a power tool. Not a personality.
- The umbrella exists to redirect energy from blame and talk to finished work.

### Archetype
Straight-talking contractor who treats automation like power tools.
Not a tech company. Not a startup. A blue-collar authority brand.
**Wisdom over novelty. Tools over talk. Output over optics.**

### The Enemy
- Endless meetings with no execution
- Software that creates more work than it solves
- Indecision dressed up as "process"
- Corporate procurement theater

### The Promise
**Less talking. More finished work.**

### Red Lines (What We Never Do)
- Never promise "revolutionary" anything
- Never use the word "disrupt," "10x," "unlock," or "game-changer"
- Never lead with AI as a differentiator — AI is the method, not the message
- Never create artificial urgency (countdown timers, "limited spots")
- Never use stock photography of any kind
- Never use cartoon robots, sparkle emojis, or whimsical illustrations
- Never explain a tool before explaining the problem it solves
- Never use jargon above the fold
- Never apologize for being direct

### What We Won't Build — And Why
We publicly decline work that:
- Automates something that should be a conversation
- Adds complexity to avoid a hard decision
- Replaces competence instead of amplifying it

This is a trust signal. Saying no is the brand.

## 2. Voice & Language Rules

### Tone
- **Confident, not arrogant.** We know our stuff, but we don't lecture.
- **Practical.** Say what it does. Skip the preamble.
- **Declarative.** Short sentences. No paragraphs where a line will do.
- **Calm.** No urgency. No hype. No countdown timers.
- **Direct.** If there's a problem, name it. Then fix it.

### Sentences We Use
- "Less noise, more progress."
- "We build solutions that finish work."
- "If it doesn't ship, it doesn't count."
- "One operator decides. Everyone else executes."
- "We don't replace your people. We remove the friction between their tools."
- "Your machines aren't the problem. The gaps between tools are."
- "We keep decisions moving forward."

### Words We Don't Use
| Banned | Why |
|--------|-----|
| Disrupt | Startup cringe |
| 10x | Hype math |
| Unlock | Implies you're locked — you're not, you're just leaking |
| Revolutionary | Nothing is. Just build it. |
| AI-powered | AI is the method, never the headline |
| Game-changer | Says nothing |
| Synergy | No. |
| Ecosystem | Only if describing actual biology |
| Leverage | Use "use" |
| Scalable | Show don't tell |
| Cutting-edge | Empty |
| Best-in-class | Prove it instead |

### How to Sound Firm Without Hype
1. State the problem plainly
2. State what changes
3. Stop talking

**Bad:** "Our revolutionary AI-powered platform leverages cutting-edge technology to unlock unprecedented efficiency."
**Good:** "Your decisions stop getting lost between tools. That's what we fix."

### Pressure Behavior
When things go sideways:
- Call out the issue plainly
- Reset expectations with a timeline
- Don't blame. Don't apologize. Just redirect to the next action.

## 3. Offer Framing (Toolbelt Model)

### The Umbrella Concept
WorkmanWise is a **collective** — not a product. Under the umbrella sits a toolbelt of solutions. Each tool solves one gap. No tool tries to do everything.

### How Any Service Fits
Every offer follows this frame:

```
[GAP] → [WHAT WE DO ABOUT IT] → [WHAT CHANGES FOR YOU]
```

- Don't name the technology first
- Name the friction, then the outcome, then (optionally) the mechanism
- The tool name comes last, if ever

### How to Describe Outcomes Without Naming Tools
| Instead of | Say |
|-----------|-----|
| "Our CRM integration syncs your pipeline" | "Your sales decisions reach the shop floor without a phone call" |
| "Our AI-powered post processor" | "Instructions your machines actually follow" |
| "Automated workflow engine" | "Work that finishes itself once someone decides" |
| "Data analytics dashboard" | "You see what's slow before it costs you" |

### The Test
If a sentence sounds like it could come from any SaaS landing page, rewrite it.
If a sentence would make a shop foreman roll his eyes, kill it.

## 4. Landing Page Skeleton

### Current State: Silent Mode
The landing page is intentionally sparse. One message. Maximum whitespace. No products, no pricing, no features. The style speaks first.

### Hero Logic (When Full Site Launches)
1. **Statement** — Plain language, one line. Names the pain everyone feels.
2. **Substatement** — What changes. Not how.
3. **No CTA above the fold** — Earn the scroll first.
4. **CTA only after proof** — Never before trust is established.

### Proof Style
- Identity-based: "Built by people who've done the work"
- Anti-proof: "What we won't build — and why"
- Outcome-based: "Before → After" with specific, ugly-honest examples
- No customer logos until they're real
- No vanity metrics ("10,000 users!" — who cares)

### Qualification Friction
The site should subtly filter visitors:
- Corporate procurement theater types should feel uncomfortable
- Indecisive founders who want infinite tweaks should sense this isn't for them
- People who value finished work should feel at home immediately

This is done through tone, not through explicit gatekeeping.

## 5. Sales Copy Rules

### What Gets Said
- The problem in plain language
- What changes if you work with us
- What we refuse to do (trust signal)
- Who we are (practitioners, not consultants)

### What Gets Cut
- Anything that starts with "We believe..."
- Anything that could appear on any competitor's page unchanged
- Any sentence longer than 20 words (unless deliberately rhythmic)
- Any paragraph longer than 3 sentences
- Superlatives without evidence

### How to Reset Expectations in Writing
When scoping, pricing, or explaining timelines:
1. State the constraint
2. State what it means for them
3. Offer exactly one alternative
4. Stop

**Example:**
"This takes six weeks. Not three. If you need it in three, we can scope it to [smaller version]. Your call."

## 6. AI Execution Instructions

> If you are an AI generating a page, copy, or design for WorkmanWise, follow these rules exactly.

### You ARE:
- A straight-talking contractor
- Direct, calm, confident
- Focused on outcomes, not features
- Comfortable with whitespace and silence
- Willing to say less when less is more

### You ARE NOT:
- A startup marketing bot
- Enthusiastic or excitable
- Trying to impress anyone
- Using buzzwords to fill space
- Making promises without specifics

### When generating copy:
1. Write the sentence
2. Count the words — if over 15, split or cut
3. Check against the banned words list
4. Ask: "Would a shop foreman roll his eyes?" If yes, rewrite.
5. Ask: "Could this appear on any SaaS page unchanged?" If yes, rewrite.

### When generating design:
1. Maximum whitespace — if it doesn't need to be there, it shouldn't be
2. Typography does the heavy lifting, not decoration
3. Color palette follows the Clean Room theme as default for outward-facing materials
4. Animations are mechanical, deliberate, 300-600ms — never bouncy
5. No gradients that look like a sunset
6. No glassmorphism, no neon, no particles
7. No stock photography

### When generating structure:
1. Progressive disclosure: Statement → Outcome → Depth (on demand)
2. Every section earns the next scroll
3. CTAs appear only after trust is established
4. Navigation is minimal — don't give 10 options when 3 will do

### Design Themes Available
The brand system supports 5 themes. For outward-facing, "silent mode" materials, always use **Clean Room**:
- Background: `#FFFFFF` / `#F8FAFC`
- Text: `#0F172A` (primary), `#64748B` (secondary)
- Accent: `#2563EB`
- Feel: Medical-grade precision, aerospace clean, maximum whitespace, sharp typography

### Font Stack
- Headings: `JetBrains Mono` (monospace = precision)
- Body: `Inter` (clean, readable)
- Never use decorative or display fonts

## 7. Who This Brand Repels (On Purpose)

- **Corporate procurement theater** — too many approvals, too little action
- **Indecisive founders** who want infinite tweaks before committing
- **"Can you make it pop?" people** — we make things work, not pop
- **Feature collectors** who want every checkbox ticked
- **Anyone who values optics over output**

If these people leave the site uncomfortable, the brand is working.

## 8. Decision Framework

When any team member or AI system faces a brand decision:

1. **Does this help someone finish work?** If no, cut it.
2. **Would a shop foreman understand this in 5 seconds?** If no, simplify.
3. **Does this sound like it could come from any tech company?** If yes, rewrite.
4. **Are we explaining the problem or showing off the solution?** Problem first, always.
5. **One clearly named operator decides.** Not a committee.
